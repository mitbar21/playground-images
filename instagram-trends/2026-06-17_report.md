## Cross-account winning themes

**Theme 1: Agent orchestration beats solo agents.** The week's dominant engagement breakout. @duncanrogoff's RuFlow post (60+ Claude Code agents, 46K/28K) is the sharpest signal: people want to understand how to run agent teams, not single agents. The structural insight — queen agents coordinate, worker agents execute, shared memory accumulates, intelligent routing cuts costs 50% — is new enough that almost no one has set it up yet. This timing gap (free tool, GitHub #1, few aware) is the content angle. Claude Fable 5's multi-day autonomous capability amplifies this: the infrastructure for sustained agent teams just got dramatically better.

**Theme 2: Claude Cowork as the one-person department.** Two separate posts from two different creators are both crushing with the same underlying mechanism: give Claude Cowork actual computer access and let it run operations. @sabrina_ramonov's marketing department (23K comments) and @advicewithjean's Instagram audit (10.5K comments) both reached their engagement peaks from different entry points — content calendar creation vs. analytics pull. The recurring pattern: /slash command + specific job to do + "comment X and I'll send you how" = outsized comment volume. The angle is not "use Claude" but "here's the specific job I gave it and what it actually did."

**Theme 3: Claude Code cost anxiety → access democratization.** @nick_saraev's free-Claude-Code post (21K/19K) and @duncanrogoff's context-loading post (37K/14K) both drove massive numbers off the same underlying pain: Claude Code is expensive to run, limits hit fast, and people are either quitting or going cautious. "Free forever" and "cut your costs 50%" are both responses to the same problem. This pain is not going away — it's worth making two or three angles from it.

**Theme 4: Claude Fable 5 as trigger event.** Anthropic launched Fable 5 on June 9. The official post (96K likes) shows the announcement format performs well at scale. The capability claims — runs for days without intervention, finance/research/law tasks, safety routing to Opus 4.8 on risky requests — are all fresh angles. Creators who can frame Fable 5 capabilities through the lens of their audience's actual blocked work ("the project you've been putting off because the AI couldn't keep up") will capture the first-mover benefit before the window closes.

**Theme 5: AI content authenticity risk.** @treysmith's "Is this real or not?" video hit 1.57M likes on a pure AI-detection angle. For any operator running AI-generated content for clients, this is the risk signal: the good stuff now looks completely real, which means the QA burden is entirely on the human layer. One viral comment thread pointing out AI tells in published client content is a trust disaster. The gap: most content operators have no explicit QA step for AI-generated output. This angle lands hardest as a Boundary Hook.

## Quick execution notes

Film these in order of confidence, not in script order. Scripts 2, 3, and 5 are the lowest-friction recordings — they're story-driven, have a clear comment trigger, and the Mitch voice comes through naturally. Scripts 1 and 8 need a graphic or visual to make the concept land — record the talking head first, then layer the visuals in editing.

For scripts that reference specific tools (RuFlow, Runway MCP, free Claude Code proxy) — do a quick five-minute setup before filming so you can speak to it from actual experience, not secondhand. If you run it yourself even once, the detail level in your delivery will be ten times sharper.

Scripts 4 and 7 are built for earned credibility moments — film them when you're in an assertive headspace, not when you're being careful. The Fable 5 script in particular needs the "no more excuses" energy to land, or it reads flat.

One comment trigger note: scripts 2, 3, 5, and 8 all have explicit triggers (COWORK, CLAUDE, COWORK again, MCP). Make sure the triggers are wired before publishing — a dead comment trigger that never delivers is a trust hit you don't need.
