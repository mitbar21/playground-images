## Cross-account winning themes

Three themes dominate this scan and they compound each other.

**Theme 1: Claude as a full operations team.** Sabrina (23,691 comments), aiagently (16,886 comments), and advicewithjean (10,354 comments) are all driving exceptional comment volume on the same core idea: Claude Cowork or Claude used as an entire marketing department, content team, or social media operation. The comment-trigger CTA mechanic is the delivery vehicle — "Comment X and I'll send the guide" converts passive viewers into warm leads at scale. This is the dominant commercial angle in the scan and the most actionable for Mitch's audience of operators and agency owners.

**Theme 2: Multi-agent stacking to extend Claude Code.** duncanrogoff's RuFlow content is the breakout of this scan — 50K likes and 31K comments on a 24K-follower account means the post spread far beyond the existing audience. The core idea, routing Claude tasks across agent tiers by complexity to cut token costs and extend usage limits, resonates hard with operators who hit Claude's usage ceiling. This angle is newer and less saturated than the "replace your team" framing.

**Theme 3: Claude beginner arbitrage.** nocode.joshua's 33,427-comment post on "3 things immediately when you start Claude" shows the ChatGPT-to-Claude migration wave is still large and largely unconfigured. The "get ahead of 90% of people" hook works because most people who switch don't set Claude up properly — memory off, no context, no skills installed. Content that bridges that gap is earning disproportionate comment volume.

**Supporting signal: AI tooling cost collapse.** nick_saraev's NVIDIA free API keys post and the open-source AI avatar content show that the AI-tools-going-free wave is real and engaging. Not core to the Claude Code / Cowork lane, but useful for hooks about removing cost barriers from AI workflows.

Delivers the payoff quantified ("cut token usage 50%, extend Claude Code 250%"). Closes on comment CTA for the setup guide.

**Cover (on-screen text):**

> Stop running one Claude session

**Script (spoken):**

Stop running all your Claude work through one session. You're burning expensive model time on simple tasks, then wondering why you hit limits before noon. The operators getting more done aren't smarter. They're routing. Complex reasoning goes to the powerful model. Repetitive execution goes to the cheap one. One agent manages the workflow. Others execute the pieces. The system shares memory and gets better over time. That's how you extend your Claude without paying more. What part of your stack is still one-model-does-everything?
